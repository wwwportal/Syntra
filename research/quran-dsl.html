1. Quran DSL

Domain-specific language and interface specifications for Quranic analysis.

2. Core Concepts

2.1. References

Quranic references use the format surah:ayah (e.g., 1:1 for Al-Fatiha verse 1).

2.2. Layers

The verse text can be explored through different linguistic layers:

  • Text - Raw Arabic text
  • Root - Triliteral root (ف ع ل)
  • Lemma - Dictionary form
  • POS - Part of speech (V, N, PREP, etc.)
  • Pattern - Morphological pattern (فَعِيل, مُفَعِّل)
  • Dependency - Syntactic relations

4. Invariants

  • surah >= 1, ayah >= 1
  • tokens[*].segments may be empty but must be present
  • Morphology entries should preserve text, pos, root, form when known
  • Dependency entries should include rel_label and word

5. Command Interface

Minimal algebraic-style laws for CLI behavior. Each command is modeled as a function on (state, output) where state = { current_verse }.

5.1. Commands

  • clear: state' = state, output' = ∅
  • see chapter n: state'.current_verse = verse(n,1), output' = render(surah n)
  • see verse a: state'.current_verse = verse(v.surah, a)
  • see verse s:a: state'.current_verse = verse(s, a)
  • layer <name|number>: Switch visible token overlay
  • layer next|prev: Navigate between layers

5.2. Rendering Invariants

  • render(verse) always includes surah:ayah and token forms/text
  • If morphology is available, includes root | pos | type | form for each segment
  • Dependency render lists relation → word (and POS when present)

6. Search Interface

6.1. Surah Search

Type the number of the surah to list all its ayat.

6.2. Ayah Search

Type the surah and ayah numbers separated by a colon. For example, 36:1 displays the first ayah of Surah Ya-Seen.

6.3. Syntactic Pattern Search

Search ayat matching specific syntactic structures.

6.4. Morphological Pattern Search

Search ayat matching morphological patterns (e.g., فَعِيل, مُفَعِّل).

6.5. Root Word Search

Search ayat containing a specific triliteral root:

root [radical1] [radical2] [radical3]

Radicals use transliterations:

  • ا = alif
  • ب = baa
  • ت = taa
  • ث = tha
  • ج = jym
  • ح = haa
  • خ = kha
  • د = dal
  • ذ = dhal
  • ر = ra
  • ز = zay
  • س = sin
  • ش = shin
  • ص = sad
  • ض = dad
  • ط = ta
  • ظ = za
  • ع = ayn
  • غ = ghayn
  • ف = fa
  • ق = qaf
  • ك = kaf
  • ل = lam
  • م = mim
  • ن = nun
  • ه = ha
  • و = waw
  • ي = ya

6.6. Phonological Pattern Search

Identify similar-sounding words with different orthographies.

6.7. Stylistic Pattern Search

Define words once; their definitions appear wherever the word is used. Undefined words show as blank boxes or Arabic text.

For example, defining "bismi" in 1:1 as 'in the symbolism of' renders:

[in the symbolism of] [allah] [ar-rahman] [ar-raheem]
